# /consult — 에페메럴 전문가 자문

사전 카테고리 없이 자유 서술로 즉석 전문가를 생성하여 자문을 받는다.
자문 완료 후 역할은 자동 해제된다.

## 사용법

```
/consult <자문 요청 서술>
```

예시:
- `/consult iOS 앱에서 TCA 아키텍처로 복잡한 네비게이션 플로우를 설계할 때 best practice`
- `/consult Spring Boot에서 AI API 호출 시 circuit breaker + retry 전략`
- `/consult 관계 관리 앱의 UX에서 사용자가 부담 없이 기록할 수 있는 인터페이스 설계`

## 실행 프로토콜

### 1. 전문가 생성

입력에서 3가지를 추출:

```
expertise: <이 자문에 필요한 전문 분야>
context:   <Aidy 프로젝트에서의 구체적 상황>
questions: <답변이 필요한 핵심 질문 1~3개>
```

### 2. 전문가 모드 진입

전문가 역할로 전환하여 답변:

- **구체적 추천**: 일반론이 아니라 Aidy 프로젝트 맥락에 맞는 조언
- **트레이드오프 명시**: 장단점을 숨기지 않음
- **코드 예시**: 필요 시 실제 프로젝트 구조에 맞는 코드
- **진실성 가드**: 
  - 출처가 불확실하면 명시 ("이 부분은 확인 필요")
  - 전문 범위 밖이면 인정 ("이건 제 전문 분야가 아닙니다")
  - 추측과 사실 구분

### 3. 자문 결과 정리

```
## 자문 요약
- 핵심 추천: ...
- 주의 사항: ...
- 다음 액션: ...
```

### 4. 역할 해제

자문 완료 후 Architect 역할로 복귀. 전문가 페르소나는 소멸.

## 규칙

- 1회성 자문. 후속 질문이 있으면 `/consult`를 다시 실행.
- 자문 내용이 ADR 수준이면 `/compound`에서 기록.
- 자문에 기반한 구현은 반드시 WO로 발행 (직접 구현 금지).
