# autoceo 폴링 토큰 최적화

**발견일**: 2026-04-19 (s26)
**상황**: 3라운드 만에 컨텍스트 ~60%를 폴링에 소비하여 조기 종료

## 증상
- 5분 간격 폴링 × 워커 평균 10분 작업 = 라운드당 폴링 6-10회
- tmux capture-pane 결과가 매번 컨텍스트에 누적
- 3라운드 후 토큰 효율 경고 → 10라운드 중 3에서 중단

## 해결 (before → after)

### 1. 폴링 간격
- Before: 5분 간격 (s6 교훈에서 설정)
- After: **20분 간격** (유저 피드백)
- 이유: 워커 평균 완료 10~15분, 20분이면 대부분 1회 폴링에 결과 확인

### 2. iOS 게이트 간소화
- Before: xcodebuild test (시뮬레이터 부팅 2분+)
- After: tuist build 통과 = 커밋 OK, xcodebuild test는 Gate-2에서만

### 3. 스펙 배치 크기
- Before: 2개/라운드 (v2.2+v2.3)
- After: 4개/라운드 (스펙만 정의하는 라운드 효율 극대화)

### 4. /compact 주기
- Before: compound 시작 시에만
- After: **매 2라운드마다** 실행

## 근본 원인
폴링은 "워커가 끝났는지 확인"하는 행위인데, 워커 완료 시간보다 폴링 간격이 짧으면 불필요한 확인이 누적된다. 최적 간격 = 워커 평균 완료 시간의 1.5~2배.
